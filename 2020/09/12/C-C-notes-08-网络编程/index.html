  • 以太网头() : 14个字节

      • 如果类型是 0806 表示后面数据为ARP请求/应答(28字节)+PAD(18字节)

socket编程主要API函数

  • Int socket(int domain, int type, int protocol);

    • 描述: 创建 socket
    • 参数:
      • domain : 协议版本
        • AF_INET : IPV4
        • AF_INET6 : IPV6
        • AF_UNIX AF_LOCAL : 本地套接字使用
      • type : 协议类型
        • SOCK_STREAM : 流式, 默认使用的协议是TCP协议
        • SOCK_DGRAM : 报式, 默认使用的是UDP协议
      • protocol : 一般用0, 表示使用对应类型的默认协议
    • 返回值:
      • 成功 : 返回一个大于0的文件描述符
      • 失败 : 返回 -1, 并设置 errno
    • 当调用socket()函数后,返回一个文件描述符,内核会提供与该文件描述符相对应的读和写缓冲区,同时还有两个队列,分别是**请求连接队列和已连接队列**.(监听文件描述符才有)

  • int bind(int sockfd, const struct sockaddr *addr, socklen_t addrlen);

    • 描述: 将socket文件描述符和 IP,PORT绑定

    • 参数:

      • sockfd : 调用socket函数返回的文件描述符

      • addr : 本地服务器IP地址和PORT

        ​ struct sockaddr_in serv;

        ​ serv.sin_family = AF_INET;

        ​ serv.sin_port = htons(8888);

        ​ //serv.sin_addr.s_addr = htonl(INADDR_ANY);

        ​ //INADDR_ANY : 表示使用本机任意有效的可用IP

        ​ inet_pton(AF_INET, “127.0.0.1”, &serv.sin_addr.s_addr);

      • addrlen : addr变量的占用的内存大小

    • 返回值:

      • 成功 : 返回 0
      • 失败 : 返回 -1,并设置 errno

  • int listen(int sockfd, int backlog);

    • 描述: 将套接字由主动态变为被动态
    • 参数:
      • sockfd : 调用socket函数返回的文件描述符
      • backlog : 同时请求连接的最大个数(还未建立连接)(最大为128, 最好填大于0的数)
    • 返回值:
      • 成功 : 返回 0
      • 失败 : 返回 -1,并设置 errno

  • int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);

    • 描述: 获得一个连接.若当前没有连接则会阻塞等待
    • 参数:
      • sockfd : 调用socket函数返回的文件描述符
      • addr : 传出参数, 保存客户端的地址信息
      • addrlen : 传入传出参数, addr变量所占内存空间大小
    • 返回值:
      • 成功 : 返回一个新的文件描述符,用于和客户端通信
      • 失败 : 返回 -1,并设置 errno
    • accept()函数是一个阻塞函数,若没有新的连接请求,则会一直阻塞. 从已连接队列中获取一个新的连接,并获得一个新的文件描述符,该文件描述符用于和客户端通信.(内核会负责将请求队列中的连接拿到已连接队列中)

  • int connect(int sockfd, const struct sockaddr *addr, socklen_t addrlen);

    • 描述: 连接服务器 (客户端调用)
    • 参数:
      • sockfd : 调用socket函数返回的文件描述符
      • addr : 服务器端的地址信息
      • addrlen : addr变量所占内存空间大小
    • 返回值:
      • 成功 : 返回 0
      • 失败 : 返回 -1,并设置 errno

  • 用 read 和 write函数可以读写(接收和发送)数据了

    • ssize_t read(int fd, void * buf, size_t count);
    • ssize_t write(int fd, const void * buf, size_t count);
    • ssize_t recv(int sockfd, void* buf, size_t len, int flags);
    • ssize_t send(int sockfd, const void* buf, size_t len, int flags);
      • 对应 recv() 和 send()这两个函数的flags直接填0就可以了
      • 注意: 如果写缓冲区已满, write()也会阻塞, read()读操作的时候,若花冲去没有数据也会引起阻塞

socket的API函数编写服务器端和客户端的步骤

  • TCP服务器端 开发流程

    socket() —-> bind() —-> listen() –+–> accept() —-> recv() —-> send() —-> close()

    1. 创建socket,返回一个监听文件描述符lfd —- socket()

      ​ —- 该文件描述符用于监听客户端连接

    2. 将 lfd和 IP PORT进行绑定 —- bind()

    3. 将 lfd由主动变为被动监听 —- listen()

    4. 接受一个新的连接,得到一个通信文件描述符cfd —- accept()

      ​ —- 该文件描述符是用于和客户端进行通信的

    5. while(1)

      {

      ​ 接收数据 —- read() 或者 recv()

      ​ 发送数据 —- write() 或者 send()

      }

    6. 关闭文件描述符 —- close(lfd); close(cfd);

  • TCP客户端开发流程

    socket() —-> connect() —-> send() —-> recv() —->close()

    1. 创建socket,返回一个通信文件描述符cfd —- socket()

      ​ —- 该文件描述符是用于和服务器通信

    2. 连接服务器端 —- connect()

    3. while(1)

      {

      ​ 发送数据 —- write() 或者 send();

      ​ 接收数据 —- read() 或者 recv()

      }

    4. 关闭文件描述符 —- close(cfd);

  • 注意:

    • 调用 accept()函数不是说新建一个连接,而是从已连接队列中取出一个可用连接

三次握手过程

  1. 客户端向服务器 发送 SYN + (随机序号1000)+(长度0)+<mss 1460(表示允许接收数据长度)>

  2. 服务器端向客户端 回复 SYN+ (随机序号8000)+(长度0) + ACK1001+<mss 1024(表示允许接收数据长度)>

  3. 客户端向服务器端回复 ACK8001

    至此,通过三次握手的连接已经建立成功了

数据发送过程

  1. 客户端向服务器端发送数据包 用前面最后一次回复过来的 1001(ACK) +(长度20)+ ACK+8001(客户端发出的最后一次ACK8001)

  2. 服务器端向客户端回复数据包 8001+ (长度10字节) + ACK 1021(是表示 1001+20个字节)

  3. 客户端向服务器发数据包 ACK8011(是表示 8001+10个字节)

    至此,数据传送部分结束

四次挥手过程

  1. 客户端向服务器端发送 FIN + 1021(刚才最后一次对方回复的1021) + (长度0) + ACK 8011

  2. 服务器端向客户端发送 ACK1022(刚才对方发的1021+1)

  3. 服务器端向客户端发送 FIN + 8011(刚才最后一次对方回复的ACK8011) + (长度0) + ACK 1022

  4. 客户端向服务器端发送 ACK8012(刚才对方发的ACK8011+1)

    至此,通过四次挥手断开连接的过程结束了

半关闭状态

此状态可以接收数据,但不能发送数据. 此状态一定是出现在主动关闭一方

  • shutdown()函数

  • shutdown()与close()之间的区别

    • shutdown()可以实现半关闭,而close()不行

    • shutdown()关闭时,不考虑文件描述符的引用计数,是直接彻底关闭.

      而close()考虑文件描述符的引用计数,调用一次close()只是将引用计数减1.只有减小到0时才会真正关闭.

  • 长连接和短连接的概念

    • 长连接: 连接建立之后,一直保持连接不关闭
    • 短连接: 连接使用完之后就立刻关闭

线程池的基本思想

​ 子线程负责从任务池中获取任务. 每一个任务有一个回调函数,每一个回调函数执行不同的操作.

    1. 先加锁,
    2. 然后判断任务池中任务是否已满?

      • 若未满, 则往任务池中添加任务,添加完任务后调用 pthread_cond_signal()通知子线程去取任务
    3. 最后解锁
  3. 子线程: 负责从任务池中获取任务并处理任务
    1. 先加锁
    2. 然后判断任务池中是否有任务?

      • 若任务池中有任务,则取任务并处理任务.
      • 处理完任务后,通知主线程继续添加任务
      • 若shutdown为1, 先解锁然后自行退出: pthread_exit;
    3. 最后释放锁

http请求消息组成部分

  • 请求行 : 说明请求类型,要访问的资源,以及使用的http版本. 如: GET /aaa.txt HTTP/1.1
    • GET : 请求指定的页面信息,并返回实体主体.GET不管信息是否涉密,都会显示出来

  • 空行 : 不能省略-而且是\r\n结尾.包括请求行和请求头,都是以\r\n结尾. 如: \r\n

http常见状态码

http状态码由三位数字组成,第一个数字代表响应的类别,有五类

  1. 1xx : 指示信息–表示请求已经接收,继续处理
  2. 2xx : 成功–表示请求已被成功接收、理解、接受
  3. 3xx : 重定向–要完成请求必须进行更进一步的操作
  4. 4xx : 客户端错误–请求有语法错误或请求无法实现
  5. 5xx : 服务端错误–服务器未能实现合法的请求

常见的状态码如下

  • 200 OK : 客户端请求成功
  • 301 Moved Permanently : 重定向
  • 400 Bad Request : 客户端请求有语法错误,不能被服务器说理解
  • 401 Unauthorized : 请求未经授权.这个状态码必须和 WWW-Authenticate报头域一起使用
  • 403 Forbidden : 服务器收到请求,但拒绝提供服务
  • 404 Not Found : 请求资源不存在
  • 500 Internal Server Error : 服务器发生不可预测的错误
  • 503 Server Unavailable : 服务器当前不能处理客户端的请求.(一段时间后可能恢复正常)
